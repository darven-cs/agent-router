# Phase Quick Plan l16: TUI Bug Zhipu Summary

## 一句话描述

主上游连续失败3次后自动切换到其他上游，通过 TrackPrimaryFailure/ShouldSkipPrimary 机制实现故障自愈。

## 修复的问题

| Bug | 描述 | 修复方式 |
|-----|------|----------|
| Bug 1 | TUI 选择上游后仍使用旧上游 | 主上游通过 SetPrimary 正确设置，proxyWithRetry 优先使用 GetPrimary() |
| Bug 2 | Zhipu 失败后下次请求未自动切换 | 连续3次失败后自动清除主上游，下次请求使用 SelectNext 切换 |

## 新增方法 (LoadBalancer)

| 方法 | 功能 |
|------|------|
| `TrackPrimaryFailure()` | 记录主上游失败，3次后自动清除主上游 |
| `ShouldSkipPrimary()` | 检查是否应该跳过主上游（基于失败计数） |
| `ClearPrimaryFailure()` | 清除失败计数（主上游成功时调用） |

## 修改的方法 (proxyWithRetry)

**逻辑流程：**

1. **请求开始**：检查 ShouldSkipPrimary()，如果主上游连续失败3次则跳过
2. **请求成功**：如果是主上游尝试，调用 ClearPrimaryFailure() 清除计数
3. **请求失败**：如果是主上游失败，调用 TrackPrimaryFailure() 累加计数
4. **连续3次失败**：主上游被自动清除，下次请求使用 SelectNext 切换到其他上游
